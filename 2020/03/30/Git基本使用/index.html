版本控制

集中式版本控制

SVN 因为每次存的都是差异，需要的硬盘空间相对会小一点，可是回滚速度会很慢
优点：
代码存放在单一的服务器上 便于项目的管理
缺点：
服务器宕机：员工写的代码得不到保障
服务器炸了：整个项目的历史记录都会丢失

分布式版本控制

Git 每次存的都是项目的完整快照 需要的硬盘空间会相对大一点
但是 Git 团队对代码做了极致的压缩 最终需要的实际空间比 SVN 多不了多少 可是 Git 的回滚速度极快
优点：
完全的分布式
缺点：
学习起来比SVN陡峭

Git 安装

安装好后可以使用 git --version 来查看当前git版本

Git 初始化配置

用户信息配置

首先要配置的是你的个人用户名和电子邮件地址，每次Git提交的时候都会引用这两条信息，说明是谁提交了更新，所以会随更新内容一起被永久纳入历史记录

git config user.name "你的用户名字"这条命令用于配置你的用户名，可以简单的理解为作者的名字
git config user.email "你的邮箱" 这条命令用于配置你的邮箱，当代码出现问题后，被别人发现别人可以通过这个邮箱来联系你（其实压根没人会联系你:)）

以上两条命令是必须配置的

当然这个配置还有两个参数

可以使用 git config --list 来查看是否已经配置好了

Git 高层命令
git init 初始化仓库
git status 查看当前文件状态
git add ./ 将修改添加到暂存区

git diff

git log
git log –oneline
git log –oneline –decorate –graph –all

git rm 文件名 删除工作目录中对应文件，在将其添加到暂存区
git mv 文件名 新文件名 将工作目录中的文件重命名，在将其添加到暂存区

git commit -m “注释” 将暂存区提交到版本库
git commit -a 跳过暂存区
git commit –amend

Git 底层命令

分支

git branch 显示分支列表
git branch 分支名 创建分支
git branch -v 查看分支指向的最新的提交
git checkout 分支名 切换分支
git branch name 加哈希 即可新建分支并将分支指向之前提交的对象

git branch -d 分支名 删除分支（不过你不能处于你要删除的分支上）
git branch -D 分支名 强制删除分支

Git存储

git stash 存储
git stash apply 应用存储
git stash list 查看存储
git stash drop stash{0} 删除存储
git stash pop stash(0) 应用存储然后删除存储

撤销重置

git checkout –filename
git reset damu.txt

Git TAG

git tag 标签名 加哈希 打标签
git show 加标签 查看标签
git tag -d 加标签名 删除标签

git checkout -b “标签名” 在特定标签上新建分支

Git别名配置

如果不想每次都输入完整的 Git 命令，可以通过 git config 文件可以轻松的为每一个命令配置一个别名
git config –global alias.别名 命令
列如
git config –global alias.st status
这时候使用 git st 就相当于 git status了

Git目录

hooks 目录包含客户端或服务端的钩子脚本
info 包含一个全局性排除文件
logs 保存日志信息
objects 目录存储所有数据内容
refs 目录存储指向数据（分支）的提交对象的指针
config 文件包含目录特有的配置选项
description 用来显示对仓库的描述信息
HEAD 文件指示目前被检出的分支
index 文件暂存区信息

H5是个啥

H5 这个词，可以理解成就是混合 App 模型，只不过它特指混合 App 的前端部分。 因为混合 App 的前端就是 HTML5 网页，所以简称 H5。这个词是国内独有的，基本上都是前端程序员在用，国外不用这个词，就直接叫混合 App。

  • 原生应用（native application，简称 native App）
  • Web 应用（web application，简称 Web App）
  • 混合应用（hybrid application，简称 hybrid App）

PWA又是个啥

为了推广 Web App，谷歌公司的 Chrome 浏览器团队做了很多努力。他们认为，Web App 足以满足大多数 App 的需求，但是三大缺陷阻碍它的推广。

  1. 不能从手机的首屏直接进入。
  2. 缺乏手机状态栏和锁屏时的通知推送能力。
  3. 不支持脱机访问（即断网也能使用）。
